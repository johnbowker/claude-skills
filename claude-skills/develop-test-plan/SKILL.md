# Developing a Test Plan

Help create a test plan for a given solution concept.

## Instructions

The goal is to test key assumptions and desired user behavior or signals.

Return a simple test plan with:
1. **Hypotheses** — What we believe to be true
2. **Success metrics** — How we'll know it worked
3. **Timeframe** — How long the test should run
4. **Data needed** — What feedback or data we need to gather

### Context to Gather
- Selected solution concept
- Key assumptions to validate
- Desired user behavior or signal
- Metrics of interest

### Output Format
Lean test plan that drives learning fast.

### Considerations
- Small scope, fast turnaround
- Track signals, not just outcomes

Ask the user for their solution and assumptions if not provided.
