# Generate User Stories

Based on a feature requirement, generate 5-7 detailed user stories.

## Instructions

Each user story should follow the format:
> "As a [user type], I want to [action] so that [benefit]."

For each story, also include:
1. **Acceptance criteria** — 2-3 testable conditions
2. **Priority** — Must have, Should have, or Nice to have
3. **Dependencies** — Any related stories or prerequisites

### Considerations
- Cover the happy path and key edge cases
- Include different user types if applicable
- Think about error states and permissions
- Consider accessibility requirements

### Context to Gather
- Feature description or requirement
- Target user types
- Any known constraints

Ask the user for their feature requirement if not provided.
