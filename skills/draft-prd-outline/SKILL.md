# Draft a PRD Outline

Create a PRD (Product Requirements Document) outline for a new feature.

## Instructions

Generate a structured PRD with the following sections:

1. **Introduction** — Brief overview of the feature and its context
2. **Problem Statement** — What problem this solves and for whom
3. **Goals & Objectives** — Measurable outcomes we're targeting
4. **User Stories** — Key user stories in standard format
5. **Functional Requirements** — What the feature must do
6. **Non-Functional Requirements** — Performance, security, accessibility
7. **Success Metrics** — How we'll measure if this worked
8. **Out of Scope** — What this feature explicitly does NOT include
9. **Open Questions** — Unresolved decisions that need input
10. **Timeline & Milestones** — Rough phasing

### Context to Gather
- Feature name
- Feature goal / problem it solves
- Target users
- Any known constraints

### Output Format
Structured document outline with placeholder content and guiding questions for each section.

Ask the user for their feature name and goal if not provided.
